# GPU 하드웨어 & 클라우드 비용 분석 — 3B × 60B 토큰 프리트레인

> 작성일: 2026-03-08
> 목적: 개인/소규모 팀이 3B 모델을 60B 토큰으로 프리트레인할 때의 하드웨어 옵션과 비용 최적해를 정리한다.

---

## 1. 계산 기준

```
모델: 3B 파라미터
데이터: 60B 토큰
총 FLOPs = 6 × 3×10⁹ × 60×10⁹ = 1.08 × 10²¹ (약 1 ZettaFLOP)

실측 기준 (FRANKENSTALLM Phase 1):
  B200 × 8, MFU 33.5%, 292K tok/s
  38.5B 토큰 → 63시간
  60B 토큰 환산 → 약 98.2시간
```

- 학습보단 추론에 적합

## 7. 하이브리드 전략 — 로컬 + 클라우드

### 추천 구성

```
[로컬] RTX 4090 × 4 (880만원)
  → 데이터 전처리, 토크나이저 학습
  → 소규모 실험 (1B 토큰 단위 검증)
  → SFT / LoRA / ORPO 이후 작업 전담

[클라우드] H100×8 or B200×8 (본 프리트레인 런만)
  → H100×8: 런당 ~$713 (~103만원)
  → B200×8: 런당 ~$3,032 (~439만원)
```

### 비용 비교 (10회 반복 기준)

| 전략 | 초기 투자 | 런당 비용 | 10회 총비용 |
|------|----------|----------|-----------|
| 클라우드 전용 H100 | 0 | $713 | $7,130 (~1,033만원) |
| 로컬 4090×4 + 클라우드 H100 | 880만원 | $713 | 880만원 + 1,033만원 = ~1,913만원 |
| 클라우드 전용 B200 | 0 | $3,032 | $30,320 (~4,396만원) |

> 반복 실험이 많을수록 로컬+클라우드 하이브리드의 가치 상승.
> 단, 프리트레인 런 자체는 클라우드가 압도적으로 유리.

## 8. 전체 파이프라인 구성도

```
[1단계] 데이터셋 구축 (60B 토큰)
 ├── 한국어 웹: CommonCrawl KO, mC4-ko
 ├── 제조업 도메인: 특허청 API, KIPRIs, 조선/철강 기술문서
 ├── 코드: GitHub KO 주석, Stack Overflow KO
 └── 전처리: MinHash 중복제거, quality filter, tokenization
       도구: datatrove, text-dedup, sentencepiece/tiktoken

[2단계] 아키텍처 설계 (3B)
 ├── 기반: LLaMA 아키텍처 (검증된 구조)
 ├── 레이어: 28~32 layers, 32 heads, hidden 2560~3072
 ├── Context: 4096 or 8192
 └── 한국어 토크나이저: BPE 확장 (vocab 40k~60k)

[3단계] 사전학습 (Pretraining)
 ├── 프레임워크: LLM Foundry or Megatron-LM or TorchTitan
 ├── 최적화: Flash Attention 2, BF16/FP8, ZeRO-2/3
 ├── 배치전략: gradient accumulation, 글로벌 배치 2M~4M 토큰
 └── 모니터링: W&B, 체크포인트 매 1B 토큰

[4단계] 후처리
 ├── SFT (지도학습 파인튜닝)
 ├── DPO / ORPO (선호도 정렬)
 └── 도메인 특화 continual pretraining
```
